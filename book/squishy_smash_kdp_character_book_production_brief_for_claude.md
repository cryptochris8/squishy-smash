# Squishy Smash — Amazon KDP Character Book Production Brief for Claude

## Project goal
Create a visually impressive, KDP-compliant paperback character book for the Squishy Smash universe.

This book should feel:
- premium
- collectible
- kid-friendly
- highly visual
- brand-building
- easy to read
- strong enough to support future expansions like stickers, coloring books, and activity books

This is not the full storybook yet.

This is the first-character-book phase:
a beautifully designed “meet the squishies” / collectible universe book.

---

# 1. Book strategy

## Recommended first book concept
Create a book in the spirit of:

**Squishy Smash: Meet the Squishies**

or

**The World of Squishy Smash**

The goal is to introduce the characters, packs, and overall universe in a way that:
- works for young readers
- looks impressive in print
- uses the assets already created
- stays achievable for KDP paperback publishing

---

# 2. Best format choice

## Recommended KDP format
Paperback
- full color interior
- full bleed
- print-ready PDF export
- visually led character book

## Recommended trim size
8.5 x 8.5 inches

Why this is recommended:
- feels premium and giftable
- works very well for character books
- gives room for strong art layouts
- fits collectible-page design well
- easier to create balanced spreads than a narrow trim

Backup option:
- 8 x 10 inches if a slightly taller page is preferred

Default recommendation:
Use 8.5 x 8.5 inches unless the user chooses otherwise.

---

# 3. KDP production rules Claude must respect

Claude must design the book around Amazon KDP paperback requirements.

## Interior format
Use a PDF interior file for print.

## Full bleed
This book should use full bleed because the art should feel immersive and premium.

When using full bleed:
- extend images/backgrounds beyond trim
- prepare the interior PDF at the bleed size, not just trim size

For KDP paperback interiors with full bleed:
- page size should be 0.125 inches wider
- page size should be 0.25 inches taller
than the selected trim size

So if trim size is 8.5 x 8.5 in, the bleed page file should be:
- 8.625 x 8.75 in

## Margin rules
For books 24–150 pages, use at minimum:
- inside/gutter margin: 0.375 in
- outside margin with bleed: 0.375 in

Claude should still leave comfortable design space and not crowd important text near trim edges.

## Cover rules
The paperback cover must be a single full-wrap cover file:
- back cover
- spine
- front cover
all in one continuous file

The exact cover dimensions depend on:
- final page count
- paper type
- trim size

Claude should use the KDP cover calculator/template process after final page count is locked.

## Spine text
Do not rely on spine text unless page count is large enough.
KDP only prints spine text on books with more than 79 pages.

Because this first character book may be shorter, Claude should design a cover that still works beautifully even if no spine text is used.

---

# 4. Recommended page count

## Best target range
24–40 pages

This is ideal because it:
- meets KDP minimum page count
- keeps production realistic
- allows generous layouts
- keeps print cost more manageable
- feels substantial enough to sell well

## Strong starting target
28 or 32 pages

That is probably the sweet spot.

---

# 5. Recommended book structure

Claude should build the book as a character/world introduction book, not a dense encyclopedia and not a traditional long narrative.

## Suggested structure

### Page 1
Title page

### Page 2
Copyright / publishing page / optional short dedication

### Page 3
Welcome to the world of Squishy Smash
- short intro
- simple tone
- explain that there are different packs/worlds

### Pages 4–5
Overview spread of the 3 main packs:
- Squishy Foods
- Goo & Fidgets
- Creepy-Cute Creatures

### Remaining spreads
Feature characters by pack in grouped visual spreads

### Final pages
- “Which squishy is your favorite?”
- “Find your pack”
- optional preview of future books
- optional simple activity prompt page

---

# 6. Best content format for the pages

Claude should adapt the collectible cards into book-friendly layouts rather than simply pasting cards on pages.

That means:
- use the character art as hero visuals
- simplify the info presentation
- enlarge text
- keep lots of breathing room
- avoid tiny card-style details that won’t read well in print

## Recommended per-character or per-spread content
For each featured character, use:
- character name
- pack name
- rarity
- a short 1–2 sentence intro
- 2–4 quick fun traits
- large art
- maybe a simple icon row

Example structure:
- Name
- Pack
- “Meet [Character]”
- favorite style / personality / fun ability
- short lore line

---

# 7. Recommended narrative tone

The tone should be:
- playful
- warm
- collectible
- magical
- cheerful
- easy for children

Claude should avoid:
- dark complicated lore
- overcrowded stats
- tiny hard-to-read text
- overly technical game terminology

This should feel like:
- a character worldbook
- a giftable children’s book
- a collectible introduction to the Squishy Smash universe

# 8. Visual design rules for the book

Claude should preserve the Squishy Smash brand language:
- glossy pastel look
- rounded friendly shapes
- premium toy-like finish
- soft gradients
- sparkles used tastefully
- visually rich but not cluttered

## Important
The book should feel connected to:
- the game
- the cards
- the website
- future merch

But the page design should be book-first, not “mobile UI pasted into pages.”

---

# 9. Character selection strategy

Claude has two viable routes:

## Option A — Full cast overview book
Include all 48 cards/characters in a more compact but beautiful presentation

Pros:
- comprehensive
- collectible
- strong for fans

Cons:
- each character gets less space
- may feel tighter visually

## Option B — Featured character book
Highlight the strongest characters first, grouped by pack

Pros:
- more premium feeling
- more visual breathing room
- easier to make stunning
- better first KDP product

Cons:
- not every character appears equally

## Recommended first release
Featured character book
with selected top characters from each pack, while referencing the larger universe.

This will likely make the strongest first KDP release.

---

# 10. Suggested first-book content model

Claude should strongly consider this structure:

## Intro spread
Introduce the Squishy Smash world

## Pack 1 section
Squishy Foods
- featured characters
- warm dessert/food world mood

## Pack 2 section
Goo & Fidgets
- bouncy sensory energy
- shiny goo visuals

## Pack 3 section
Creepy-Cute Creatures
- spooky-cute but sweet
- magical creature feel

## Closing spread
Choose your favorite squishy / come back for more / future adventure teaser

This is stronger than trying to fit every card equally in the first book.

---

# 11. Text density guideline

Claude should keep text light.

Recommended:
- 30 to 90 words per page
- sometimes less if the art is strong
- no overcrowding

This is a visual character book, not a chapter book.

---

# 12. Typography guidance

Claude should use:
- large readable headings
- child-friendly fonts
- strong hierarchy
- no tiny labels
- high contrast on printed page

Print readability matters more than “card authenticity.”

---

# 13. Page design guidance

Claude should design with:
- safe text zones
- strong focal point art
- simple page rhythm
- recurring section structure
- pack color coding if useful

Avoid:
- overstuffed pages
- text too close to trim
- tiny decorative clutter
- backgrounds that hurt legibility

---

# 14. Cover concept direction

The cover should feel:
- premium
- colorful
- collectible
- kid-friendly
- Amazon-search friendly

## Strong cover direction
- big Squishy Smash title
- 3–5 strongest characters on cover
- glossy magical look
- clear subtitle such as:
  - Meet the Squishies
  - A Character Adventure Book
  - The World of Squishy Smash

Claude should create a cover that works even without spine text if the page count stays below the threshold for printed spine text.

---

# 15. KDP-safe production workflow

Claude should follow this sequence:

1. lock trim size
2. lock target page count
3. decide featured-character structure
4. design interior page templates
5. build the interior at proper bleed dimensions
6. export print-ready PDF
7. calculate exact cover using KDP cover template after final page count is known
8. export final full-wrap cover PDF or PNG per production workflow

---

# 17. What Claude should avoid

Do not:
- build the book like a full card binder
- use tiny unreadable card text everywhere
- depend on spine text
- overcrowd pages
- use layouts that ignore KDP bleed and margin rules
- overcomplicate lore
- treat the first book like a long novel

a full-color, full-bleed, visually impressive paperback character/world introduction book for Amazon KDP, using 8.5 x 8.5 trim, approximately 28–32 pages, with selected featured characters grouped by pack and a highly polished collectible Squishy Smash visual style.
